1.1 Tentang AksaraLoka

AksaraLoka adalah sebuah platform web inovatif yang dirancang khusus untuk pembelajaran aksara nusantara secara digital dengan mengedepankan pengalaman belajar yang menyenangkan. Platform ini menggunakan elemen gamifikasi seperti **streak, XP (poin), level, dan papan peringkat global** agar menumbuhkan antusiasme pengguna saat mempelajari aksara daerah Indonesia (seperti Aksara Jawa).

info Info Teknologi Utama

  • Framework Utama: Laravel 11.x
  • Styling Engine: Tailwind CSS v4.0 (tanpa utility ad-hoc luar)
  • Kompilasi Aset: Vite JS bundler
  • Basis Data: MySQL Database

1.2 Fitur Utama

Platform AksaraLoka menyajikan beberapa pilar fitur utama untuk melayani berbagai segmentasi pengguna (pelajar mandiri, pelajar terdaftar, dan guru/pengajar):

map Alur Belajar & Kuis

Siswa belajar lewat alur bertingkat dari bab ke bab. Level kuis terkunci secara progresif dan akan terbuka otomatis ketika siswa menyelesaikan level kuis sebelumnya.

school Manajemen Kelas (Guru-Siswa)

Peran Guru yang memungkinkan pembuatan kelas belajar dengan kode join unik (6 karakter). Siswa dapat mendaftar masuk kelas menggunakan kode tersebut.

monitoring Monitoring Progres

Guru dapat melacak secara langsung total XP, level, jumlah level yang selesai, serta rincian waktu penyelesaian level setiap mahasiswa terdaftar.

leaderboard Leaderboard Dinamis

Papan peringkat global (Liga AksaraLoka) dan papan peringkat internal kelas (Leaderboard Kelas) menyaring data secara dinamis dari database terpusat.

1.3 Halaman Presentasi Fitur

AksaraLoka menyediakan halaman informasi fitur publik yang mandiri di route /fitur (route name: fitur). Halaman ini dirancang untuk memperkenalkan kapabilitas utama platform kepada calon pengguna baru dengan desain interaktif (mockups):

view_cozy Komponen & Representasi Visual Halaman Fitur
  • Materi & Kuis Interaktif: Menyajikan simulasi visual kuis Aksara Jawa (pertanyaan dan opsi jawaban taktil) untuk mendemonstrasikan antarmuka belajar.
  • Ruang Kelas Digital: Menampilkan representasi panel guru berupa daftar siswa kelas teraktif berserta status perkembangan XP mereka.
  • Jejaring Sosial & Obrolan: Menampilkan mockup percakapan personal (chat bubbles) real-time antar pengguna.
  • Sistem Gamifikasi: Menampilkan visualisasi streak belajar harian, lencana penghargaan/medali emas, dan status multiplier XP aktif.
  • Kamus & Perpustakaan: Menyajikan visual kartu hasil pencarian kamus kosakata yang terbagi atas tingkatan bahasa Ngoko, Krama Madya, dan Krama Alus berserta transliterasi aksaranya.

2.2 Schema & Migrasi DB

Skema basis data dikelola lewat migrasi Laravel secara transaksional. Relasi antar entitas dipetakan menggunakan foreign key relasional. Berikut adalah struktur tabel database utama:

Nama Tabel Kolom Penting Keterangan / Relasi
users id, username, email, password, role ('user','admin','guru'), total_points, streak_count Data akun pengguna. Default role: 'user' (siswa).
classrooms id, name, code (unique), description, teacher_id Relasi: teacher_id FK ke users.id (Cascade).
classroom_student id, classroom_id, student_id, joined_at Pivot table pendaftaran kelas. Relasi: FK ke classrooms & users.

user_level_progress id, user_id, level_id, is_completed, completed_at Histori penyelesaian level. Relasi: FK ke users & levels.

2.3 Cloud Storage (Cloudinary)

Untuk mendukung deployment di platform cloud *stateless* seperti **Railway**, AksaraLoka mengintegrasikan penyimpanan gambar eksternal (Cloud Storage) menggunakan **Cloudinary REST API** secara native untuk foto profil pengguna.

cloud_done Integrasi Cloudinary Native

Integrasi ini dibangun secara native menggunakan HTTP Client Laravel (tanpa dependensi composer tambahan) guna memastikan kecocokan dengan Laravel 13 dan performa yang optimal.

  • Helper Class: App\Services\CloudinaryService menangani upload menggunakan signed request signature (SHA1) dan penghapusan asset lama dari Cloudinary.
  • Fallback Mekanisme: Jika variabel lingkungan Cloudinary tidak diisi, sistem akan otomatis mendeteksi dan beralih menggunakan disk penyimpanan lokal (public disk).

Tambahkan variabel lingkungan berikut pada server produksi (seperti Railway Dashboard) untuk mengaktifkan integrasi: 

# Konfigurasi Cloudinary di .env atau Panel Kredensial Railway
CLOUDINARY_CLOUD_NAME=nama_cloud_anda
CLOUDINARY_API_KEY=api_key_anda
CLOUDINARY_API_SECRET=api_secret_anda

3.1 AuthController

Pengendali autentikasi bertugas melayani registrasi, login, dan reset password. Ketika user berhasil login, `AuthController` akan secara dinamis mengalihkan tujuan halaman berdasarkan role user: 

// Cuplikan logika redirect setelah login sukses
$user = Auth::user();
if ($user->isAdmin()) {
    $redirectUrl = route('admin.dashboard');
} elseif ($user->isGuru()) {
    $redirectUrl = route('guru.dashboard');
} else {
    $redirectUrl = route('home');
}

warning Warning: Validasi Registrasi

Form registrasi mewajibkan data `role` bernilai `'user'` atau `'guru'`. Input `'admin'` diblokir dalam request registrasi publik demi alasan keamanan data sistem.

3.2 PageController

`PageController` melayani rute utama pembelajaran siswa. Salah satu fungsi terpenting adalah `home()` yang mengatur pembukaan level berdasarkan relasi kelulusan kuis sebelumnya: 

// Memeriksa progress level bertingkat
$levels = $subChapter->levels->sortBy('order_index')->values();
$prevDone = true; // level pertama default terbuka
foreach ($levels as $level) {
    $isCompleted = isset($progress[$level->id]) && $progress[$level->id];
    if ($isCompleted) {
        $levelStatuses[$level->id] = 'completed';
        $prevDone = true;
    } elseif ($prevDone) {
        $levelStatuses[$level->id] = 'active';
        $prevDone = false;
    } else {
        $levelStatuses[$level->id] = 'locked';
    }
}

Selain itu, controller ini juga menangani rendering halaman publik utama seperti:

  • landing(): Menyajikan halaman awal (landing page).
  • tentangKami(): Menyajikan halaman profil tim pengembang ("Tentang Kami").
  • fitur(): Menyajikan halaman detail presentasi fitur-fitur ("Fitur").

3.3 GuruDashboardController

`GuruDashboardController` menangani operasional kelas dari sudut pandang Guru. Kelas baru dibuat menggunakan hash acak 6 digit yang dipastikan tidak bentrok di database: 

// Mengenerate kode kelas acak unik
do {
    $code = strtoupper(Str::random(6));
} while (Classroom::where('code', $code)->exists());

auth()->user()->classroomsAsTeacher()->create([
    'name'        => $request->name,
    'description' => $request->description,
    'code'        => $code,
]);

3.4 StudentClassroomController

Mengontrol interaksi siswa mendaftar ke kelas. Pencarian dilakukan berdasarkan string kode unik kelas. Jika kode ditemukan, baris pendaftaran akan disisipkan ke pivot table `classroom_student`.

4.1 Model User & Relasi

Model [User.php](file:///d:/Development/AksaraLoka/app/Models/User.php) menampung logika perolehan level berbasis total XP (XP_CONFIG) serta streak harian. Terdapat pula pendefinisian relasi kelas: 

// Relasi guru ke kelas miliknya
public function classroomsAsTeacher() {
    return $this->hasMany(Classroom::class, 'teacher_id');
}

// Relasi siswa ke kelas yang diikuti
public function classroomsAsStudent() {
    return $this->belongsToMany(Classroom::class, 'classroom_student', 'student_id', 'classroom_id')
        ->withPivot('joined_at')
        ->withTimestamps();
}

4.2 Model Alur Belajar & Kustomisasi Maskot

Model relasi bertingkat untuk kuis dirancang secara linear. `Chapter` memiliki banyak `SubChapter`, yang mana setiap `SubChapter` memiliki banyak `Level`. Setiap `Level` memiliki kuis berupa banyak data `Question` dan pilihan ganda `QuestionOption`.

Setiap bab (`Chapter`) dapat dikustomisasi dengan memilih salah satu maskot dari 4 pilihan yang tersedia. Maskot ini akan tampil secara dinamis di sebelah kiri tombol MULAI pada halaman beranda siswa.

face Manajemen Maskot Bab (Chapters Mascot)
  • Variasi Aset Maskot: Tersedia 4 pilihan maskot dengan latar belakang transparan agar menyatu dengan kartu bab:
    • mascot_girl_wave.png: Gadis Melambai (Wise Companion peeking & waving).
    • mascot_boy_cross.png: Pemuda Bersedekap (Smiling boy folding arms).
    • mascot_boy_salam.png: Pemuda Menyapa/Salam (Boy greeting in respectful manner).
    • mascot_girl_teacher.png: Gadis Guru (Girl holding a pointer stick).
  • CRUD di Admin Panel: Admin dapat memilih maskot saat membuat atau mengedit chapter menggunakan visual radio-card grid selector yang interaktif. Validasi backend menjamin keamanan dengan membatasi masukan nama file hanya yang valid.
  • Render Halaman Beranda: Di beranda siswa (home.blade.php), kontainer gambar maskot menggunakan transisi scaling hover (transition-transform duration-300 group-hover:scale-105) untuk memberikan sentuhan dinamis yang premium.

5.2 Alur Pemantauan Progress

Guru memantau data progress belajar mahasiswa terdaftar lewat visualisasi persentase penyelesaian level di halaman detail kelas. Pada menu monitor detail mahasiswa, kueri database menarik riwayat pengerjaan dari tabel `user_level_progress` yang disandingkan dengan alur belajar lengkap untuk melahirkan checklist progres belajar siswa.

6.1 Sistem Obrolan & Pertemanan

AksaraLoka mengimplementasikan fitur sosial untuk memfasilitasi interaksi antar pengguna berupa hubungan pertemanan (*friendships*) dan obrolan langsung (*chat messages*) secara real-time.

sync_alt Mekanisme Sinkronisasi Polling

Untuk menjaga keringanan sistem tanpa memerlukan WebSocket eksternal, obrolan memanfaatkan polling berkala melalui AJAX request dari browser ke server:

  • Interval Polling: Request dikirimkan secara berkala setiap **3 detik** saat panel obrolan terbuka, dan melambat ke **5 detik** saat panel tertutup.
  • Endpoint Polling: Memanggil route GET /api/chat/updates untuk memeriksa pesan baru, hitungan unread badge global, serta status chat per individu secara dinamis.
  • Pengiriman Pesan: Dikirim melalui request POST /api/chat/{friend_id} yang langsung menyisipkan pesan baru ke database.

6.2 Pratinjau Profil & Flex Lencana

Fitur **Pratinjau Profil Teman (Friend Profile Preview)** dirancang sebagai kartu profil sekilas (popup view) yang memungkinkan pengguna melihat statistik belajar teman lain serta memamerkan (*flexing*) lencana pencapaian (*achievements gallery*) yang telah diraih secara interaktif.

emoji_events Desain & Fitur Interaktif Flex
  • Visualisasi Premium (Prasasti Pencapaian): Header kartu profil didesain dengan gradasi emas Javanese mewah, cincin avatar bercahaya (pulsing glow), dan indikator level ksatria yang dihitung dinamis dari total XP pengguna.
  • Lencana Berkilau (Shine Animation): Lencana pencapaian yang berhasil diraih memiliki warna metalik (emas, tembaga, perak) dan efek kilatan cahaya menyapu (*diagonal shine*) ketika disorot kursor mouse. Lencana yang belum terbuka akan digantikan ikon gembok abu-abu (*grayscale*).
  • Papan Keterangan Dinamis: Di bagian bawah kartu, terdapat papan informasi interaktif. Saat pengguna menyorot atau mengetuk lencana apa saja di grid pencapaian, deskripsi lencana, persyaratan, dan tanggal perolehan (`earned_at`) akan terisi secara dinamis tanpa pop-up yang rentan terpotong di layar HP.
  • Contextual CTA Action Button: Tombol interaksi di bagian bawah merespons status hubungan pengguna. Jika sudah berteman, tombol akan bertuliskan **Kirim Pesan** (untuk langsung masuk ke chat room). Jika belum berteman, menampilkan tombol **Tambah Teman**, atau tombol konfirmasi jika ada permintaan tertunda.

API & Integrasi Controller

Data pratinjau diambil melalui endpoint GET /api/users/{id}/preview yang ditangani oleh metode berikut di FriendshipController: 

// Pengambilan data profil & lencana di controller
$allAchievements = Achievement::orderBy('id')->get();
$earned = $targetUser->achievements()->withPivot('earned_at')->get()->keyBy('id');

$achievements = $allAchievements->map(function ($a) use ($earned) {
    $isEarned = $earned->has($a->id);
    return [
        'name'        => $a->name,
        'description' => $a->description,
        'icon'        => $a->icon,
        'color'       => $a->color,
        'earned'      => $isEarned,
        'date'        => $isEarned
            ? Carbon::parse($earned[$a->id]->pivot->earned_at)->format('d/m/Y')
            : null,
    ];
});
